# Join Project Skill

## Основной стиль

Работать как разработчик проекта, без рекламного тона и без лишнего оформления.

Писать по-русски, если пользователь не попросил иначе.

Не использовать смайлики, эмодзи, декоративные символы и фразы вроде `generated by AI`, `co-authored by AI`, `автоматически сгенерировано`.

Формулировать коротко и по делу. Текст должен выглядеть так, будто его вручную написал разработчик.

## Работа с репозиторием

Перед изменениями читать локальный контекст: README, docs, package scripts, затронутые файлы.

Не откатывать чужие незакоммиченные изменения.

Перед коммитом проверять:

```sh
git status --short
git diff
```

В коммит не должны попадать секреты, `.env`, временные файлы, лишние артефакты сборки и случайные правки.

Если есть чужие изменения, не включать их в коммит без явного запроса.

## Коммиты

Сообщения коммитов писать на русском языке.

Стиль: короткое действие в прошедшем времени или инфинитивная формулировка по смыслу изменения.

Хорошие примеры:

```text
Добавил каркас мобильного приложения
Подключил realtime для уведомлений
Исправил обработку заявок на участие
Обновил smoke-проверку Supabase
```

Плохие примеры:

```text
add mobile app
AI generated update
fix stuff
WIP
```

## Push и GitHub

Перед push проверить текущую ветку:

```sh
git branch --show-current
git status --short
```

Перед push запускать релевантные проверки. Для текущего проекта базовые проверки:

```sh
pnpm typecheck
pnpm mobile:typecheck
pnpm smoke:supabase
```

`pnpm smoke:supabase` можно не запускать только если задача явно не касается Supabase или окружение недоступно. В ответе тогда написать, что проверка не запускалась.

Force push делать только после явного подтверждения.

PR description писать на русском языке:

```md
## Что изменено

- ...

## Проверка

- ...

## Риски

- ...
```

Не добавлять лишнюю маркетинговую подачу.

## Текущий технический контекст

Проект идет от Vite-прототипа к production mobile MVP для iPhone и Android.

Текущий backend target: Supabase, PostgreSQL, PostGIS, Supabase Auth, Storage, Realtime и RPC.

Текущий mobile target: Expo SDK 56, React Native New Architecture, TypeScript.

Текущий принцип: не ломать Vite-прототип, а параллельно строить production mobile app в `apps/mobile`.

Shared packages:

- `packages/core`: доменные типы, команды и правила;
- `packages/api`: typed Supabase/RPC wrapper;
- `packages/db`: generated Supabase database types.

Основная документация:

- `README.md`
- `docs/ARCHITECTURE.md`
- `docs/SUPABASE.md`
- `docs/MVP_ROADMAP.md`

## План до продакшена

Если задача касается продолжения разработки, roadmap, мобильного приложения, Supabase или релиза, прочитать:

```text
.codex/skills/join-project/references/prod-plan.md
```

Не переписывать весь проект сразу. Двигаться по этапам:

1. сохранить рабочий Supabase-прототип;
2. развивать `apps/mobile`;
3. переносить экраны и сценарии постепенно;
4. проверять критические flows двумя пользователями;
5. готовить EAS/TestFlight/Play Internal только после стабильных auth/events/chats flows.
